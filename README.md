# ESP32-IntelliCtrl

> 基于 ESP32-S3 的智能文件中枢：蓝牙配网、WiFi 剪贴板、无线文件服务器三合一
---
## 主要特性

- **BLE 蓝牙快速配网**：通过手机 App 完成 WiFi 配置，无需传统串口/显示屏连接
- **WiFi 剪贴板同步**：多设备（手机/电脑）快速复制粘贴文本
- **无线文件服务器**：SD 卡/TF 卡远程上传、下载、删除
- **本地菜单界面**：支持ST7789液晶屏实时显示、本地按键控制
- **丰富的娱乐/工具**：内置小游戏与函数画板等实用工具
- **高度可定制**：所有硬件引脚均可通过 `gpio.json` 动态配置

## 简介

ESP32-IntelliCtrl 面向日常办公、生活等多场景文件传输 & 管理问题，用户摆脱数据线、U盘、云盘工具，实现无线、跨平台的文件互通。基于 MicroPython，无需第三方库，轻松移植及二次开发。

---

## 快速开始

1. **准备硬件：**  
   ESP32-S3开发板 + 1.9寸ST7789屏幕 + TF卡模块 + 按键/LED  
   （如有N20等型号请根据实际引脚适配）

2. **固件烧录：**  
   - 下载[MicroPython固件](https://micropython.org/download/esp32/)
   - 使用烧录工具（如 esptool.py、烧录软件）将固件写入开发板

3. **上传项目源码：**  
   - 复制本项目中 `/workspace/` 所有 `.py` 文件至开发板

4. **引脚配置（可选）：**  
   - 如需要自定义引脚，请编辑 `gpio.json`

5. **蓝牙配网：**  
   - 打开配套App，搜索并连接设备，填写WiFi信息，完成无线配网

6. **功能体验：**  
   - 浏览器访问：`192.168.4.1`（AP模式）或设备分配的局域网地址（STA模式）

## 硬件配置

| 功能         | GPIO引脚 | 说明           |
| ------------ | -------- | -------------- |
| TFT_SCL      | GPIO40   | SPI 时钟信号   |
| TFT_SDA      | GPIO45   | SPI 数据信号   |
| TFT_CS       | GPIO42   | 屏幕片选       |
| TFT_DC       | GPIO41   | 数据/命令选择  |
| TFT_RST      | GPIO39   | 屏幕复位       |
| TFT_BL       | GPIO46   | 背光控制(PWM)  |
| SD_CLK       | GPIO14   | SD卡 SPI时钟   |
| SD_CMD       | GPIO15   | SD卡数据输出   |
| SD_D0        | GPIO16   | SD卡数据输入   |
| SD_CS        | GPIO21   | SD卡片选       |
| KEY          | GPIO0    | 按键输入       |
| LED          | GPIO1    | 状态灯         |

硬件可通过 `gpio.json` 个性化配置。

## 协议与 API

### BLE 蓝牙协议

- **行业标准服务：**  
  - 采用 Nordic UART Service，UUID: `6E400001-B5A3-F393-E0A9-E50E24DCCA9E`
  - 特征值：0xFF01（READ | WRITE | NOTIFY）
- **连接配对：**  
  - 通常无须手动配对，连接后自动通讯。
- **BLE 事件：**  
  - 连接/断开/写操作后，触发状态处理和指令下发。

#### 主要数据指令

- **WiFi配网指令**  
  格式一（新协议，推荐）：
  ```
  [0xCC 0x05 CMD 0x00 LEN Payload... CHK]
  ```
  校验和 = 包体除 CHK 以外所有字节之和 & 0xFF  
  - 例如：
    - 添加WiFi（分两步Token机制）  
      1. [0x39] ADD: `[0xCC 05 39 00 LEN [SSID_LEN][SSID][TOKEN] CHK]`
      2. [0x3C] SET_PWD: `[0xCC 05 3C 00 LEN [TOKEN][PWD_LEN][PWD] CHK]`
    - Token 有效期 30 秒，SSID 支持最多 12字节。  
    - 删除、切换索引、获取列表全兼容旧协议。

  - 兼容旧协议格式：
    ```
    [0xCC 0x05 CMD 0x00 0x00 CHK]
    ```
    校验等，详见对应指令业务分发。

- **遥控指令（0xBB系列）**  
  ```
  [0xBB 0x05 [CMD] 0x00 0x00 CHK]
  ```
  支持上/下/左/右/OK/BACK等遥控按键。

- **壁纸/屏幕指令（0xAA系列）**  
  壁纸切换：
  ```
  [0xAA 0x03 CMD ID 0x00 CHK]
  CMD: 0xA0=图片背景, 0xB0=纯色背景
  ID: 背景编号
  ```
  屏幕设置：
  ```
  [0xAA 0x04 CMD VAL 0x00 CHK]
  CMD: 0xC0=开关, 0xC1=亮度, 0xC3=息屏时间
  VAL: 参数值
  ```

- **状态查询和通知**
  - 查询：[0x34]
  - 主动通知：`0x35`，内容包含模式、连接状态、IP、SSID、文件数量、SD卡状态等，所有蓝牙事件（连接变更、状态刷新）均会自动推送。

#### 通用数据分包协议（配置文件传输）

- **数据通道命令字：** 0xDD
- **数据包格式：**
  ```
  [0xDD DATA_TYPE SEQ_H SEQ_L LEN ...PAYLOAD]
  ```
  - DATA_TYPE: 0x02=api.json, 0x03=gpio.json，可扩展
  - SEQ_H/SEQ_L: 包编号（0起始，16位），SEQ=0为首包
  - LEN: 有效数据字节数
  - PAYLOAD: 数据内容

- **分包重组：**
  - 支持自动多包重组，内容完整后保存至设备端文件。
  - 接收完整包后自动回传ACK。

- **会话控制：**
  - 支持多会话（类型区分），每会话最多 30秒超时。
  - 旧会话超时后自动清理，避免资源泄露。

#### 注意事项

- 所有 WiFi 指令必须按新协议优先，旧协议仅兼容历史设备。
- 错误/非法命令会有详细日志与特殊响应码（如0xFE、0xFF）。
- 全过程支持 Notify 主动推送以及响应，请开启手机端通知功能获取实时状态。

---

详细协议交互、应用层示例可参考 `bluetooth_ctrl.py` 实现。

- **WiFi 配网 / 文件管理等协议举例：**  
  - 配网指令格式：`0xCC 0x05 0x30 ... 校验和`
  - 使用HTTP进行文件上传/下载操作，接口兼容标准Web请求。

## 常见问题

1. **连接不上设备？**  
   - 检查串口是否被占用、引脚配置是否一致、烧录是否完整。

2. **传输速度慢？**  
   - 确认WiFi信号良好，建议SD卡为高速型号。

3. **如何扩展新功能？**  
   - 参考 `main.py` 模块化结构，自定义功能模块并注册即可。
