# ESP32-IntelliCtrl
ESP32-S3智能文件中枢 - 蓝牙配网、WiFi剪贴板、无线文件服务器三合一  一个基于ESP32-S3的微型智能设备，通过BLE蓝牙实现快速配网， 内置WiFi剪贴板跨设备同步，支持浏览器无线访问SD卡文件。  核心功能： • BLE WiFi配网 - 无需屏幕，手机直配 • WiFi剪贴板 - 复制即同步 • WiFi U盘 - 浏览器无线文件管理  硬件：ESP32-S3 | 显示屏：ST7789 | 存储：TF卡 语言：MicroPython
 ESP32-IntelliCtrl
一款基于 ESP32-S3 的智能文件中枢，支持蓝牙配网、WiFi 剪贴板、文件传输三大核心功能。

项目简介
 ESP32-IntelliCtrl是一款专为 ESP32-S3 开发板设计的全能型文件中枢软件。它将微型的嵌入式设备转变为功能丰富的智能控制中心，用户可通过蓝牙和 WiFi 与设备进行交互，实现文件传输、剪贴板同步、屏幕控制等操作。整个项目采用 MicroPython 开发，代码简洁易懂，便于二次开发和定制。

从功能定位来看， ESP32-IntelliCtrl瞄准的是日常办公和生活中常见的文件传输场景。传统的文件传输方式往往需要数据线、U 盘或云存储等中介，操作繁琐且受网络环境限制。 ESP32-IntelliCtrl通过将 ESP32-S3 打造成一个便携式无线文件服务器，用户只需通过浏览器即可访问 SD 卡中的文件，无需安装任何客户端应用。同时，蓝牙配网功能让设备的网络配置变得前所未有的简单，即使是没有屏幕的嵌入式设备，也能轻松完成 WiFi 连接。

在技术实现上， ESP32-IntelliCtrl充分利用了 ESP32-S3 的双核处理器、大容量内存和丰富的外设接口。BLE 蓝牙用于接收控制指令和配置信息，WiFi 模块提供网络连接和 HTTP 服务，SPI 接口驱动显示屏和 SD 卡。整个系统的设计遵循模块化原则，各功能模块独立运行又相互协作，便于维护和扩展。

核心功能
 ESP32-IntelliCtrl提供三大核心功能，分别解决不同的使用场景需求。这三个功能既可以独立使用，也可以组合使用，形成完整的无线文件管理解决方案。

蓝牙配 WiFi 功能让用户无需通过串口或屏幕即可完成网络配置。用户打开配套的手机 APP，通过 BLE 连接设备后，选择目标 WiFi 并输入密码，设备即可自动连接。设备会记住成功的配置，下次开机时自动重连，无需重复配网。这一功能特别适合没有显示屏或调试接口的嵌入式设备，大大降低了部署难度。

WiFi 剪贴板功能实现了跨设备的剪贴板同步。用户在不同设备间复制粘贴文本时，无需借助微信、邮件或云笔记等中介工具。只需确保手机和电脑连接在同一局域网内，通过浏览器访问设备 IP 即可查看和同步剪贴板内容。该功能采用实时推送机制，文本更新后立即同步到所有已连接的设备，真正做到了“复制即同步”。

WiFi U 盘功能将 ESP32-S3 变身为一个便携式无线文件服务器。用户可以通过浏览器上传、下载、删除 SD 卡中的文件，传输速度可达 300KB/s。

硬件配置
 ESP32-IntelliCtrl项目的硬件平台为 ESP32-S3 开发板，配合 ST7789 显示屏和 SD 卡存储模块。以下是详细的硬件配置说明。

主控芯片采用乐鑫科技的 ESP32-S3，这是一款基于 Xtensa LX7 双核处理器的 SoC 芯片，主频可达 240MHz，内置 512KB SRAM 和 4MB SPI Flash。ESP32-S3 支持 2.4GHz WiFi 和蓝牙 5.0（BLE），非常适合需要无线连接功能的物联网项目。在本项目中，ESP32-S3 负责运行 MicroPython 解释器、处理蓝牙协议栈、管理 HTTP 服务器、控制显示屏和读写 SD 卡。

显示屏采用 1.9 英寸 ST7789 IPS 液晶屏，分辨率为 172x320 像素。该屏幕采用 SPI 接口与 ESP32-S3 通信，仅需 6 根信号线即可驱动。ST7789 支持 16 位色彩深度，显示效果清晰艳丽，适合展示动画和图形界面。屏幕的背光由 GPIO 控制，支持 5%-100% 的亮度调节。

存储模块使用 TF 卡（MicroSD 卡），通过 SPI 接口连接。SD 卡作为文件的实际存储介质，在文件传输过程中，系统采用分块传输策略，每传输 32KB 数据执行一次内存回收，确保大文件传输的稳定性。SD 卡的文件系统格式支持 FAT32，这也是 MicroPython 默认支持的文件系统格式。

所有硬件引脚均可通过 gpio.json 配置文件自定义，无需修改代码即可适配不同的硬件连接方式。以下是默认的引脚配置：

TFT_RST	GPIO39	屏幕复位

SD_CMD	GPIO15	SD 卡数据输出

SD_CS	GPIO21	SD 卡片选
KEY	GPIO0	按键输入
LED	GPIO1	状态指示灯
软件架构
 ESP32-IntelliCtrl项目采用模块化设计，各功能模块职责明确，通过标准接口相互调用。这种设计使得代码易于维护和扩展，也为开发者提供了清晰的二次开发路径。

项目根目录下包含以下核心文件：
main.py 是系统的入口文件，负责初始化各模块并启动主循环；
config.py 集中管理所有可配置参数，包括引脚定义、协议常量、显示配置等；
bluetooth_ctrl.py 实现 BLE 蓝牙协议栈，处理所有蓝牙通信；
wifi_mgr.py 管理 WiFi 连接和配网逻辑；
screen_ctrl.py 控制屏幕的开关、亮度和息屏功能。

文件服务相关模块包括：
base_server.py 定义文件服务器的公共基类，包含 HTTP 解析、URL 编解码、安全检查等通用方法；
file_server.py 实现 AP 模式的完整文件服务器，提供文件上传、下载、删除功能；
file_server_sta.py 实现 STA 模式的轻量级服务器，仅提供剪贴板同步功能。文件服务器的设计采用策略模式，根据 WiFi 模式自动选择合适的服务器类型。

用户界面相关模块包括：
display_ctrl.py 提供状态栏、传输进度等 UI 元素的绘制函数；
display_menu.py 实现菜单系统的渲染和交互逻辑；
menu_def.py 定义菜单结构和各菜单项的行为；
file_browser.py 实现文件浏览器的界面和操作逻辑。这些模块共同构成了设备的本地用户界面，用户可以通过按键直接操作设备。

游戏和工具模块包括：
snake_game.py 实现经典的贪吃蛇游戏；
graph_board.py 提供函数图像绘制工具，支持二维和三维图形的动态演示。这些模块丰富了设备的娱乐功能，让用户在工作之余可以使用设备进行休闲放松。

配置和工具模块包括：
settings_mgr.py 管理用户设置的持久化存储；
wallpaper_mgr.py 控制屏幕背景的显示；
time_sync.py 实现网络时间同步功能；
key_scan.py 实现按键扫描和去抖逻辑。这些模块为系统提供辅助功能支持。

BLE 蓝牙协议
 ESP32-IntelliCtrl通过 BLE 蓝牙与手机 APP 进行通信，协议设计遵循简洁高效的原则。设备作为 GATT 服务器，提供一个 Nordic UART Service，特征值为 0xFF01，支持读、写、通知三种操作。手机 APP 通过向该特征值写入指令数据，设备执行相应操作后通过通知返回结果。

协议帧格式
蓝牙指令采用变长帧格式，不同类型的指令有不同的帧结构。以下是协议的基本约定。

所有指令帧均以帧头开始，帧头固定为 0xAA（用于系统控制指令）或设备定义的其他前缀。帧头后跟随长度字节，表示从帧头到校验和之前的数据总字节数。不同指令的长度字节有不同的含义，部分指令的长度字节与后续字节配合表示协议版本或子命令类型。数据段包含指令的实际参数，不同指令有不同的参数定义和顺序。帧尾为校验和字节，使用前 N 字节之和的低 8 位作为校验值，接收方通过校验和判断数据完整性。

系统控制指令（0xAA 前缀）
系统控制指令用于壁纸切换、屏幕控制、状态查询等设备管理功能。该类指令的帧头为 0xAA，长度字节表示从帧头到校验和前的数据长度。

壁纸切换指令用于更换设备的显示背景。图片背景切换的帧格式为：帧头 0xAA，长度 0x03，命令 0xA0，图片编号 0x01-0x08，保留字节 0x00，校验和。纯色背景切换的帧格式为：帧头 0xAA，长度 0x03，命令 0xB0，颜色编号 0x01-0x10，保留字节 0x00，校验和。设备收到壁纸切换指令后，返回确认响应：帧头 0x00（成功）或错误码，命令字节，参数字节，保留字节 0x00。

屏幕控制指令用于控制屏幕的开关、亮度和息屏时间。屏幕开关指令的帧格式为：帧头 0xAA，长度 0x04，命令 0xC0，开关值（0=关闭，1=开启），保留字节 0x00，校验和。亮度调节指令的帧格式为：帧头 0xAA，长度 0x04，命令 0xC1，亮度值（5-100），保留字节 0x00，校验和。息屏时间指令的帧格式为：帧头 0xAA，长度 0x04，命令 0xC3，超时值（0=永不息屏，10=10秒，30=30秒，60=60秒，300=5分钟），保留字节 0x00，校验和。

屏幕状态查询指令用于获取屏幕的当前状态。查询指令的帧格式为：帧头 0xAA，长度 0x04，命令 0xB0，保留字节 0x00，保留字节 0x00，校验和。设备响应的数据格式为：帧头 0xAA，长度 0x06，命令 0xB0，屏幕开关（0=关闭，1=开启），当前亮度（5-100），息屏超时（秒），校验和。

WiFi 控制指令（0xCC 前缀）
WiFi 控制指令用于管理设备的网络连接，包括切换 WiFi 模式、扫描网络、保存配置等操作。该类指令的帧头为 0xCC，适用于需要双向确认的 WiFi 操作。

WiFi 模式切换指令用于在 STA、AP、关闭三种模式间切换。切换到 STA 模式的指令格式为：帧头 0xCC，长度 0x05，命令 0x30，保留字节 0x00，保留字节 0x00，校验和。设备收到指令后开始连接保存的 WiFi，连接成功返回：响应码 0x00，连接状态（0x01=已连接），信号强度（dBm）。切换到 AP 模式的指令格式为：帧头 0xCC，长度 0x05，命令 0x32，保留字节 0x00，保留字节 0x00，校验和。关闭 WiFi 的指令格式为：帧头 0xCC，长度 0x05，命令 0x33，保留字节 0x00，保留字节 0x00，校验和。

WiFi 扫描指令用于获取周围可用的无线网络列表。扫描指令的格式为：帧头 0xCC，长度 0x05，命令 0x36，保留字节 0x00，保留字节 0x00，校验和。设备开始扫描后，首先返回扫描结果头：命令 0x36，网络数量，低位字节数，高位字节数，保留，保留。然后逐个返回各网络的详细信息：命令 0x36，序号，SSID 长度，SSID 内容（UTF-8 编码），RSSI 值，安全性标志（0x00=开放，0x01=加密）。

WiFi 列表管理指令用于操作保存的 WiFi 配置列表。获取列表指令的格式为：帧头 0xCC，长度 0x05，命令 0x38，保留字节 0x00，保留字节 0x00，校验和。设备返回已保存的 SSID 列表：命令 0x38，SSID 数量，然后是每个 SSID 的 10 字节定长数据（不足补零）。添加 WiFi 指令的格式为：帧头 0xCC，长度 0x05，命令 0x39，SSID 长度，SSID 内容，PWD 长度，PWD 内容，校验和。删除 WiFi 指令的格式为：帧头 0xCC，长度 0x05，命令 0x3A，索引，保留字节，校验和。连接指定 WiFi 指令的格式为：帧头 0xCC，长度 0x05，命令 0x3B，索引，保留字节，校验和。

遥控指令（0xBB 前缀）
遥控指令用于模拟按键操作，实现对设备本地界面的远程控制。该类指令的帧头为 0xBB，指令格式统一为：帧头 0xBB，长度 0x05，命令字节，保留字节 0x00，保留字节 0x00，校验和。

支持的方向键命令包括：0x10 表示上键，0x11 表示下键，0x12 表示左键，0x13 表示右键，0x14 表示确认键（OK），0x15 表示返回键（BACK）。设备收到遥控指令后，执行相应的按键动作并返回操作结果。成功响应格式为：帧头，成功标志，命令字节，保留字节。失败响应格式为：错误码。

数据传输协议（0xDD 前缀）
数据传输协议用于传输大型数据，如配置文件、剪贴板内容或文件数据。该协议支持分包传输，单包最大 20 字节，可传输任意大小的数据。

数据包的帧格式为：命令 0xDD，数据类型，会话 ID 高位，会话 ID 低位，负载长度，负载数据。数据类型 0x02 表示 API 配置，0x03 表示 GPIO 配置。首包的会话 ID 为 0，后续包使用相同的会话 ID 以标识同一数据流。设备收到数据包后立即写入 SD 卡，采用追加模式，支持流式传输大文件。

错误码定义
蓝牙通信过程中可能遇到各种错误情况，设备通过错误码告知手机 APP 错误类型，以便进行相应的处理。以下是完整的错误码定义：

错误码	含义	说明
0x00	成功	操作执行成功
0x01	失败	操作执行失败
0xF1	索引错误	指定的 WiFi 索引超出范围
0xF2	文件错误	JSON 文件读写失败
0xF3	SSID 为空	添加 WiFi 时 SSID 为空
0xF4	Token 无效	配网 Token 不存在或已过期
0xF5	无配置	没有保存的 WiFi 配置
0xFC	未知命令	收到不支持的指令
0xFD	校验错误	数据校验和不匹配
0xFE	格式错误	指令格式不符合协议规范
0xFF	系统错误	发生未预期的系统异常
HTTP API 接口
 ESP32-IntelliCtrl在 WiFi 连接成功后启动 HTTP 服务器，提供基于浏览器的文件管理界面和 API 接口。AP 模式下服务器监听 192.168.4.1:80，STA 模式下服务器监听局域网 IP:80。
